Purpose & Aim

With the Cookie Control module you will have the ability to show a pop up in your Zen Cart shop asking your visitors permission to use cookies.

Features

  • Display a pop-up at th bottom of your pages
  • All necessary settings can be set through the admin panel
  • Configure minor lay-out changes trough the admin panel

Installation

!!!!!BACKUP YOUR FILES AND DATABASE BEFORE YOU BEGIN!!!!!

BEFORE INSTALLING, you are strongly encouraged to READ ALL OF THE TABS in this readme file. Then follow the installation or upgrade instructions, as appropriate.

BEFORE YOU POST A QUESTION ON THE FORUM - read through this readme again as needed

There are two folders in the file set

  • 1_install_files
  • 2_core_edits
New Installation Instructions
  1. BACK UP your database & store.
  2. Unzip the Cookie Control package to your local hard drive, retaining the folder structure.
  3. Rename the "YOUR_ADMIN_FOLDER" folder in the "1_install_Files"
    • 1_install_files\YOUR_ADMIN\
  4. Rename all of the "YOUR_TEMPLATE" folders to match the name of your template folder.
    • 1_install_files\includes\templates\YOUR_TEMPLATE\
    • 2_required_core_edits\includes\templates\YOUR_TEMPLATE\
  5. Upload the files from "1_install_files" to your store. (DO NOT upload the "1_install_files" folder, just the CONTENTS of this folder -
    copy ALL of the add-on files to your store!! Most issues are caused by store owners who decide to NOT load ALL of the module files)
  6. Upload the files from "2_required_core_edits" to your store. (DO NOT upload the "2_required_core_edits" folder, just the CONTENTS of this folder - copy ALL of the add-on files to your store!! Most issues are caused by store owners who decide to NOT load ALL of the module files)
    • If the file in the "2_required_core_edits" folder has been modified by other module installs, you will need to use a file merging program like Beyond Compare or WinMerge to incorporate the Cookie Control changes
  7. Optional: Check if you are running at least jQuery 1.4.4 on your server. If so, you may proceed to step 8. If not, copy the contents of the 3_optional_files folder to your server (By default the script will start jQuery from Google if it isn't already on your server.)
  8. Once the files are uploaded, click ANY link in your store's admin, and the auto-installer will be triggered. If you have followed the install instructions, you will see the installation confirmation message.
  9. Remove your Google Analytics code from the cart, Cookie Control will take Control.
  10. Now you are ready to start configuring Cookie Control
  11. For other cookies to be controlled by Cookie Control add a function to tpl_cookie_control.php, by 'copying' the Analytics function. For detailed help see CIVIC website.

Un-Install Instructions
  1. BACK UP your database & store.
  2. Delete the following from your current install:
    • YOUR_ADMIN/includes/autoloaders/config.cookie_control.php
    • YOUR_ADMIN/includes/extra_datafiles/cookie_control.php
    • YOUR_ADMIN/includes/functions/extra_functions/cookie_control.php
    • YOUR_ADMIN/includes/init_includes/init_cookie_control_config.php
    • YOUR_ADMIN/includes/installers/cookie_control/ (complete folder)
    • YOUR_ADMIN/includes/languages/YOUR_LANGUAGE/extra_definitions/cookie_control.php
    • includes/templates/YOUR_TEMPLATE/stylesheet_cookie_control.css
    • includes/templates/YOUR_TEMPLATE/jscript/jscript_cookieControl-5.1.min.php
    • includes/templates/YOUR_TEMPLATE/templates/tpl_cookie_control.php
    • includes/languages/YOUR_LANGUAGE/extra_definitions/YOUR_TEMPLATE/cookie_control.php
  3. Remove the changes you made to includes/templates/YOUR_TEMPLATE/common/tpl_main_page.php
  4. Run the uninstall.sql from the "Sql files" folder

Configuration

  1. Go to Configuration => Cookie Control Configuration in the admin panel

    Enable Cookie Control
    Enable or disable Cookie Control (enabled by default)

    Choose your Consent Model
    Select the consent model you wish Cookie Control to use. In each consent model the Cookie Control panel appears to the user when they first access the site.
         

    Information Only

    Implicit

    Explicit
    Informs users the site is using cookies. They are given no option to opt out. Informs users the site is using cookies and they are given the option to opt out. Informs users the site would like to use cookies and they are given the option to opt in.
    Information Only Implicit Explicit

    Choose your icon

    Position your icon

    Choose your theme
    The icon will be anchored to the corner of the viewport and won't scroll with your web page. Choose whether you would like your icon to appear on the left or right of the browser window. Choose whether you would prefer the Cookie Control widget to have a light or dark display.
    Diamond Left Light
    Triangle Right Dark

    Countries for showing Cookie Control
    Please enter the list of countries for which you wish the plug-in to appear. If left blank Cookie Control will appear for all users from all countries. The full list of countries can be found here
     

    Pop up by default
    Choose whether the Cookie Control user interface (UI) is open by default on page load. This makes it much more explicit that you're seeking user's consent for the use of cookies, and may be a safer option in terms of compliance, until Cookie Control icon is widely recognised.
     

    Hiding Cookie Control after consent
    Choose whether you want to hide the Cookie Control icon after the user has given consent then modify the onAccept callback.
     

    Google Analytics
    If you use Google Analytics, enter your Google Analytics key here. This will cause Cookie Control to interact with Google Analytics, opting users in or out depending on their preference.
     

    Apply to sub-domains
    If you have multiple sub-domains, you can choose whether Cookie Control opt-ins apply across all sub-domains or just a single domain.

    E.g. if your main site is at www.mysite.com and your blog is at blog.mysite.com, you'd probably want a single opt-in to apply across both sites. If you select "No", users will have to give there consent once on each site.

Troubleshooting

Basics

Make sure your custom template is active. (Admin > Tools > Template Selection)

Make sure Cookie Control is installed. Admin => Configuration => Cookie Control Configuration

If Cookie Control does not work or gives you errors:

  • Make sure all files are in correct location
  • Make sure you uploaded ALL the Cookie Control files
  • Make sure the files are not corrupt from bad FTP transfers
  • Make sure your file merge edits are correct
  • MAKE SURE YOU RE-READ THE CONFIGURATION AND USAGE SECTIONS!!!

Files

Cookie Control makes only one minor modification to core Zen Cart files in order to work correctly. If you've installed other mods in your store, you should use a program like WinMerge or Beyond Compare to merge in the required change for Cookie Control into your existing store files. If this is a fresh (vanilla) Zen Cart install, you can simply overwrite the default Zen Cart files.

Here is the full list of core Zen Cart files that are affected by Cookie Control:

  • includes/languages/YOUR_TEMPLATE/YOUR_LANGUAGE.php (no merging necessary as of v1.1)
  • includes/templates/YOUR_TEMPLATE/common/tpl_main_page.php
